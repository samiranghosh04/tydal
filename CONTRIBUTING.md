# Contributing to Tydal

Thank you for your interest in contributing to Tydal! We welcome contributions from the community.

## How to Contribute

### Reporting Issues

- Use the GitHub issue tracker to report bugs or request features
- Provide detailed steps to reproduce the issue
- Include your environment details (OS, Expo version, device)

### Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/tydal-period-tracker.git`
3. Install dependencies: `npm install`
4. Start the development server: `npx expo start`
5. Make your changes
6. Test thoroughly on both iOS and Android
7. Submit a pull request

### Code Style

- Follow the existing code style and conventions
- Use TypeScript for all new code
- Run `npx expo lint` to check for linting issues
- Ensure all tests pass

### Pull Request Guidelines

- Create a descriptive title and description
- Reference any related issues
- Keep PRs focused on a single feature or fix
- Ensure your code doesn't break existing functionality
- Add tests for new features when possible

### Commit Messages

- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove)
- Keep the first line under 50 characters
- Add more details in the body if needed

## Development Guidelines

### Security

- This app handles sensitive health data
- Never commit sensitive information or credentials
- Follow security best practices for data handling
- Report security issues privately to the maintainers

### Testing

- Test on multiple devices and platforms
- Verify biometric authentication works correctly
- Test data export functionality
- Ensure offline functionality works properly

### Documentation

- Update README.md for any new features
- Add JSDoc comments for new functions
- Keep documentation current
